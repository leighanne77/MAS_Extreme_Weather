## Data Access Limitations

### **Proprietary Data We Cannot Access**
- Individual company financial performance data
- Internal risk models and algorithms
- Customer relationship management data
- Insurance premium calculations and coverage details
- Supply chain cost structures and vendor relationships
- Due diligence reports and environmental assessments
- Partner company financials and operational data
- Internal performance metrics and KPIs

### **Regulatory/Government Data We Cannot Access**
- Classified government procurement information
- Internal agency planning documents
- Federal infrastructure funding allocations
- State-level economic development incentives
- Local permitting timelines and requirements
- Individual taxpayer information
- IRS Form 8997 filings and compliance data

### **Market Data We Cannot Access**
- Competitor investment strategies
- Real estate valuations for specific properties
- Labor market costs for specialized skills
- Equipment and technology pricing
- Individual investor capital gains amounts and timing
- Internal fund structures and investor agreements

### **Clarification: Referenced vs. Integrated APIs**

- **Referenced Data Sources ([Ref])**: These are listed for user awareness and future integration. There is no code in the Pythia system that directly connects to these APIs yet.
- **Integrated APIs ([API])**: These have a corresponding Python module in `src/multi_agent_system/data/` (e.g., `openet_api.py`, `fred_api.py`, `erddap_mcp.py`).
- **MCP Providers ([MCP])**: These are accessed via the Model Context Protocol, with native provider modules in the codebase (e.g., `erddap_mcp.py`, `datagov_mcp.py`, `cmr_mcp.py`).

**Best Practices:**
- When referencing a data source, provide a link and a short description, but do not imply direct integration unless a code module exists.
- For new integrations, create a dedicated module in `src/multi_agent_system/data/` and update the table to [API] or [MCP] as appropriate.
- See the project documentation and `src/multi_agent_system/data/` for current integrations and usage examples.

## Data Loader Error Handling, Logging, and Enum Usage

All non-API/non-MCP data loaders in Pythia use robust error handling and logging, following system-wide rules (see `docs/00_cursor_rules.md`).

- All file I/O, parsing, and transformation steps are wrapped in try/except blocks.
- The system logger is used for error, warning, and info messages.
- Custom exceptions and error objects are used as required.
- Error messages are clear and actionable for both agents and developers.

### Enum Usage in Loader Return Values

Loader and agent return values use the following enums for standardization and agent compatibility:
- `DataLoadStatus` (success/error)
- `ProvenanceType` (api/mcp/static/manual)
- `DataSourceType` (agriculture/water/economic/environmental/other)
- `UpdateFrequency` (realtime/hourly/daily/weekly/monthly/annual/manual)
- `DataFormat` (json/csv/xml/parquet/other)
- `ErrorType` (network/parsing/validation/permission/unknown)
- `AccessLevel` (public/restricted/private/internal)
- `ArtifactType` (dataset/model/report/visualization/log)

**Rationale:**
- Ensures robust, traceable, and agent-compatible error handling and data provenance.
- Enables agents to programmatically interpret loader results, error types, and data freshness.
- Follows conventions in `docs/00_cursor_rules.md` and system architecture.

See also: `docs/Definitions.md` for a full list of enums and their values.

### ADK/A2A Tool Integration

All data loaders are exposed as ADK tools via:
1. **`@adk_tool` decorator** in `loader_tools.py` for metrics collection
2. **AgentCards** in `agents/cards.py` for discoverability
3. **Standardized return values** using enums from `src/enums.py`

### Return Value Structure

All loaders return a standardized dict:
```python
{
    "status": DataLoadStatus.SUCCESS | DataLoadStatus.ERROR,
    "data": <response data or None>,
    "provenance": DataProvenance.API | DataProvenance.STATIC | ...,
    "update_frequency": DataUpdateFrequency.ANNUAL | ...,
    "data_format": DataFormat.JSON | ...,
    "access_level": DataAccessLevel.PUBLIC | ...,
    "domain": DataDomain.ECONOMIC | DataDomain.AGRICULTURE | ...,
    "error_type": DataErrorType.NETWORK | None,
    "error": <error message or None>,
    "metrics": <LoaderMetrics dict>,
}
```
