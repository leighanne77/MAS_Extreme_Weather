## Overview

Tool is an AI-powered multi-agent system that provides extreme weather risk assessment and nature-first resilience recommendations for capital market actors. The system transforms complex extreme weather data into actionable insights for investment, lending, and risk management decisions.

This is a sophisticated multi-agent system that's essentially a stateful, memory-enabled wrapper around LLMs, following solid software engineering principles. The agents are really just specialized microservices that handle different aspects of data processing, memory, and state management. Our system can provide complex risk assessments and recommendations - because it is not just the LLM doing the work, but the entire software infrastructure built around it that makes it useful and reliable:

- **Take user input**: Handled by `src/multi_agent_system/agents/base_agent.py` and `src/multi_agent_system/communication.py`
- **Process it through specialized components (agents)**: Orchestrated by `src/multi_agent_system/agent_team.py` and `src/multi_agent_system/coordinator.py`
- **Maintain state across interactions**: Managed by `src/multi_agent_system/session_manager.py` and `src/agentic_data_management/data_manager.py`
- **Finds and format data for LLM consumption**: Processed by `src/multi_agent_system/data/data_loader.py` and `src/multi_agent_system/data_management.py`
- **Store results and context**: Handled by `src/agentic_data_management/agents/lifecycle_agent.py` and `src/multi_agent_system/artifact_manager.py`
- **Handle security and compliance**: Managed by `src/agentic_data_management/agents/security_agent.py` and `src/agentic_data_management/governance.py`

## System Overview

### **Core Purpose**
Tool addresses the critical need for extreme weather-related risk assessment in financial decision-making by providing:
- **Location-specific risk analysis** for extreme weather events
- **Nature-based adaptation strategies** with ROI analysis frameworks (no guarantees - statistically significant and measurable improvements only)
- **Multi-agent intelligence** combining specialized expertise
- **7 specialized user types** tailored for different professional roles (Primary prototype: Private Equity Investor)

**Decision Support Tool**: Pythia is a decision support tool, NOT a decision making tool. It cannot be automated into any systems and does not integrate into Private Equity banks or other financial services systems. Users export analysis results and integrate them into their own proprietary systems as needed.


The system serves 7 (lucky) specialized user types:
1. **Private Equity Investors** - Asset protection and ROI analysis
2. **Private Debt Managers** - Can be from private equity firms offering this  
3. **Chief Risk Officers** - Portfolio-level risk management for banks and financial services firms
4. **Financial Services Industry Chief Sustainability Officers** - Seeking to push logical, high ROI-driven risk mitigations and provide their firms with unique market differentials  
5. **Financial Services Indusry Data Science Officers** - Model validation and data integration
6. **Operating Credit Officers** - Seasonal credit and working capital management
7. **Government Funders** - Including Federal/central and local and examples include rural development officers, urban infrastructure planning, and disaster response managers

## High-Level Architectural Overview

### **System Architecture Layers**

The system follows a **layered architecture** with clear separation of concerns and specialized components for each functional area:

#### **Frontend Layer** 🌐
- **Web Interface (FastAPI)**: Main user interface with role-based customization
- **User Onboarding**: Role selection and preference management for 7 specialized user types
- **Query Interface**: Natural language processing with intent recognition
- **Results Display**: Interactive visualizations using Chart.js and responsive design
- **Export Functions**: Multi-format export (PDF, Excel, JSON) for stakeholder integration

#### **Integration Layer** 🔗
- **Natural Language Processing**: Query parsing and intent recognition for user queries
- **User Type Management**: Role-specific customization and interface adaptation
- **Session Management**: State persistence and user preference tracking
- **Data Formatting**: Response formatting optimized for different user types
- **A2A Protocol**: Complete Agent-to-Agent communication with message routing

#### **Agent Layer** 🤖
- **Risk Agent**: Analyzes extreme weather risks with confidence scoring (see `src/multi_agent_system/agents/risk_agent.py`)
- **Historical Agent**: Provides historical context and trend analysis (see `src/multi_agent_system/agents/historical_agent.py`)
- **News Agent**: Integrates current events and recent information (data refresh: 1-6 hours depending on source) (see `src/multi_agent_system/agents/news_agent.py`)
- **Recommendation Agent**: Suggests nature-based adaptation strategies (see `src/multi_agent_system/agents/recommendation_agent.py`)
- **Validation Agent**: Cross-validates results and ensures data quality (see `src/multi_agent_system/agents/validation_agent.py`)
- **Greeting Agent**: User onboarding for PE investors (see `src/multi_agent_system/agents/greeting_agent.py`)
- **Farewell Agent**: Session closure and summary (see `src/multi_agent_system/agents/farewell_agent.py`)
- **Base Agent**: Base agent class with ADK features (see `src/multi_agent_system/agents/base_agent.py`)
- **Cards**: Agent card definitions (see `src/multi_agent_system/agents/cards.py`)
- **Tools**: Agent tools for risk analysis (see `src/multi_agent_system/agents/tools.py`)
- **Data Management Agents**: 20 specialized agents for data governance and quality (see `src/agentic_data_management/agents/`)

#### **Data Layer** 📊
- **NOAA Weather Data**: Historical and forecast weather data integration
- **Nature-Based Solutions**: Comprehensive database with case studies, cost-benefit analysis, and implementation guidance (see nature_based_solutions.json)
- **Historical Records**: Past extreme weather events and impact analysis (see historical_weather_events.json)
- **Economic Impact Data**: Financial analysis frameworks and ROI analysis (see economic_impact_data.json)
- **Enhanced Data Sources**: International and specialized datasets (see enhanced_data_sources.py)
- **Regional Data**: Prototype-specific data for all geographic regions (see regional_risk_profiles.json)
- **Data Refresh Intervals**: 1-6 hours depending on source (not real-time)
- **Additional Data Files**: Biodiversity_Credit_Revenue_Streams.json, funding_sources_NSB.json

#### **Infrastructure Layer** ⚙️
- **Google ADK**: Agent Development Kit for multi-agent orchestration (see `src/multi_agent_system/adk_integration.py`)
- **FastAPI**: High-performance Python web framework
- **SQLite/PostgreSQL**: Database for artifacts and session data
- **Redis**: Caching and session management
- **Performance Monitoring**: System health and performance tracking (see `src/multi_agent_system/performance/` directory: benchmarking.py, caching.py, load_testing.py, monitoring.py, optimization.py)
- **Configuration Management**: System configuration (see `src/multi_agent_system/config.py`)
- **Workflow Management**: Workflow definitions (see `src/multi_agent_system/workflows/workflows.py`)
- **Utility Functions**: ADK feature utilities (see `src/multi_agent_system/utils/adk_features.py`)

### **Key Architectural Principles** 🎯
- **API-First Design**: RESTful API for programmatic access. Export-based integration only - users export analysis results and integrate them into their own proprietary systems. No direct system connections.
- **Function-Based Tools**: Python functions automatically wrapped by ADK
- **Event-Driven Architecture**: Asynchronous processing with retry logic
- **Mobile-First Responsive Design**: Works across all device types
- **Security by Design**: Comprehensive authentication and validation
- **Scalable Architecture**: Horizontal scaling with caching and optimization
- **Decision Support Architecture**: Export-based integration only - no direct system connections

### **Data Flow** 🔄
1. **User Input**: Natural language query through web interface or API
2. **Query Processing**: NLP parsing and intent recognition
3. **Agent Coordination**: A2A protocol routing to specialized agents
4. **Data Analysis**: Multi-agent analysis with confidence scoring
5. **Result Synthesis**: Integration of agent outputs and formatting
6. **Response Delivery**: Interactive visualization and export options
