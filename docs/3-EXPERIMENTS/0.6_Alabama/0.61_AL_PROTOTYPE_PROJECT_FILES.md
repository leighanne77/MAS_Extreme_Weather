## Overview

This document lists all project files needed to build a prototype focused on the Alabama Private Equity Investor user. The prototype addresses extreme weather risk assessment for manufacturing facilities in Mobile Bay, Alabama, with a focus on Opportunity Zone investments.

## Agent Search Behavior and System Rules

Agents are instructed to use all curated JSON files (such as `nature_based_solutions.json` and `coastal_areas_infrastructure_needs.json`) as starting points only. They must conduct external research to find current, location-specific, and expert knowledge data sources, and validate all data before use. See docs/_RULES_Pythia_System_Rules/ and docs/00_cursor_rules.md for details on agent autonomy, data validation, and modularity.

> **Note:** More data sources and integrations will become available over time as the system evolves and new APIs are added.

### **Data Files - Local Government & Expert Knowledge Data Sources**

**Consolidated Approach**: Instead of multiple separate JSON files for state/regional data, use two unified files with examples from multiple regions and organization types.

```
src/multi_agent_system/data/
├── government_data_sources_examples.json        # Examples of government data sources (local, state, federal/national)
└── local_expert_knowledge_data_sources_examples.json  # Examples of expert knowledge sources (non-profits, community, academic, industry)
```

**Purpose**: These JSON files provide curated examples of data sources for extreme weather risk assessment. Agents use these as starting point examples and must conduct external research to find location-specific data sources and current information.

**Geographic Coverage**: Examples include:
- Mobile Bay, Alabama, USA (state agencies, Mobile Baykeeper, fishing unions)
- Gulf Coast, USA (general)
- South India (Tamil Nadu, Kerala state agencies, local organizations, village panchayats)
- Other global coastal regions

**Government Data Sources** (`government_data_sources_examples.json`):
- **State/Provincial**: State environmental agencies, commerce departments, transportation agencies, emergency management
- **Federal/National**: EPA, NOAA, FEMA, USGS (USA); Central Pollution Control Board, Indian Meteorological Department (India)
- **Local/Municipal**: City planning departments, county emergency management, municipal corporations, village panchayats

**Expert Knowledge Data Sources** (`local_expert_knowledge_data_sources_examples.json`):
- **Non-Profit Organizations**: Environmental NGOs (e.g., Mobile Baykeeper), community organizations, advocacy groups
- **Community Organizations**: Fishing unions (e.g., Mobile Bay fishing unions), industry associations, local cooperatives
- **Academic Research**: Research institutions, universities, scientific organizations (e.g., Dauphin Island Sea Lab, ICAR)
- **Industry Associations**: Port authorities, chambers of commerce, trade organizations
- **Indigenous Knowledge Holders**: Traditional knowledge keepers, indigenous communities (with cultural protocol notes)

**Data Categories**:
- Environmental data (water quality, air quality, ecosystem health)
- Economic data (development trends, investment data, industry information)
- Infrastructure data (transportation, building codes, construction regulations)
- Emergency management data (disaster preparedness, response plans, risk assessments)
- Local knowledge (traditional observations, community knowledge, indigenous knowledge)

**Key Design Principles**:
- Examples from multiple regions (not just Mobile Bay, Alabama)
- Includes non-profit and community organizations (e.g., Mobile Baykeeper, Mobile Bay fishing unions)
- Includes India examples (state agencies, local organizations, village panchayats)
- Includes indigenous knowledge holders with cultural protocol guidance
- Agents use as starting points and conduct external research for current, location-specific sources
- Uses "extreme weather" terminology (not "climate")
- Focuses on decision support (not decision-making)
- Includes access methods and best practices for data source research
- Notes relationship building requirements for community and indigenous knowledge sources

## Agent Directory Structure and Roles

The following agents are implemented in `src/multi_agent_system/agents/`:
- `base_agent.py`: Base agent class with ADK features
- `risk_agent.py`: Risk analysis agent (hurricanes, heat, flooding)
- `historical_agent.py`: Historical weather event analysis
- `recommendation_agent.py`: Risk mitigation strategy recommendations
- `validation_agent.py`: Cross-validation of risk assessments
- `greeting_agent.py`: User onboarding for PE investors
- `farewell_agent.py`: Session closure and summary
- `news_agent.py`: News and information agent
- `reference_data_agent.py`: Manages and curates reference data JSONs; develops governance rules for reference data in dialogue with users; researches metadata sources and finds authoritative data sources for all use cases
- `query_refinement_agent.py`: Helps users clarify and refine queries for more accurate analysis and data retrieval
- `due_diligence_agent.py`: Supports due diligence by validating data sources, checking compliance, and flagging missing or questionable information
- `progress_display_agent.py`: Displays progress and status updates to users during long-running analyses
- `cards.py`: Agent card definitions
- `tools.py`: Agent tools for risk analysis

Each agent interacts with the data source modules as needed, using the [API] and [MCP] integrations above. Agents are designed to be modular and extensible, following the system rules for autonomy and data validation.
